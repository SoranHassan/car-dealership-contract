# سرور اشتراک/لایسنس AutoGarideh

این سرور سه کار انجام می‌دهد:
1. نگه‌داری حساب هر مشتری (نام کاربری/رمز، تاریخ انقضای اشتراک، فعال/غیرفعال).
2. پنل وبی مدیریت در مسیر `/admin` (با یک رمز ادمین) برای افزودن مشتری، تمدید اشتراک،
   فعال/غیرفعال کردن آنی، روشن/خاموش کردن بک‌آپ خودکار، و دانلود بک‌آپ‌های دریافتی.
3. API که برنامه‌ی دسکتاپ هر بار اجرا (و به‌صورت دوره‌ای) با آن چک می‌کند تا مطمئن شود
   اشتراک هنوز معتبر است.

## دیپلوی روی Railway (پیشنهادی، ساده‌ترین راه)

1. یک ریپوی گیت‌هاب جدا برای همین پوشه‌ی `server/` بسازید (یا کل پروژه را push کنید؛
   Railway از روی `server/Dockerfile` می‌فهمد این پوشه یک سرویس مستقل است).
2. در [railway.app](https://railway.app) یک پروژه‌ی جدید بسازید → **Deploy from GitHub repo**
   → ریپو را انتخاب کنید → Root Directory را روی `server` بگذارید.
3. Railway به‌طور خودکار `Dockerfile` را پیدا و اجرا می‌کند.
4. یک **Volume** به سرویس اضافه کنید (Settings → Volumes) و آن را روی مسیر `/data`
   mount کنید — این‌جا دیتابیس و فایل‌های بک‌آپ ذخیره می‌شوند و بین دیپلوی‌ها از بین
   نمی‌روند.
5. متغیرهای محیطی زیر را در Settings → Variables تنظیم کنید:
   ```
   LICENSE_ADMIN_PASSWORD=<یک رمز قوی برای پنل مدیریت>
   LICENSE_SERVER_SECRET=<یک رشته‌ی تصادفی و طولانی — مثلاً با: python3 -c "import secrets; print(secrets.token_hex(32))">
   DATA_DIR=/data
   ```
6. بعد از دیپلوی، Railway یک دامنه‌ی رایگان می‌دهد (مثلاً
   `autogarideh-license-production.up.railway.app`). این آدرس را کپی کنید.
7. به `/admin` بروید و با `LICENSE_ADMIN_PASSWORD` وارد شوید.

## تنظیم آدرس سرور در برنامه‌ی دسکتاپ

قبل از تحویل برنامه به یک مشتری، در فایل `ui/license_client.py` این خط را با آدرس
واقعی سرورتان پر کنید:

```python
LICENSE_SERVER_URL = "https://autogarideh-license-production.up.railway.app"
```

⚠️ این مقدار **عمداً** در تب تنظیمات پنل ادمین برنامه قابل‌تغییر نیست — چون اگر باشد،
یک نمایندگی می‌تواند آن را به سرور جعلی خودش تغییر دهد و کل بررسی اشتراک را دور بزند.
این مقدار را فقط شما، قبل از build/ارسال هر نسخه، در کد تنظیم می‌کنید.

## ساخت اولین مشتری

1. وارد `/admin` شوید.
2. فرم «افزودن مشتری جدید» را پر کنید (نام کاربری، رمز، تعداد روز اشتراک).
3. همین نام کاربری/رمز را در اختیار مشتری بگذارید — اولین بار که برنامه را اجرا کند،
   دیالوگ ورود اشتراک این‌ها را از او می‌پرسد.

## تست محلی (بدون دیپلوی، برای توسعه)

```bash
cd server
pip install -r requirements.txt
export LICENSE_ADMIN_PASSWORD=testpass
export LICENSE_SERVER_SECRET=dev-secret
export DATA_DIR=./data
uvicorn app.main:app --reload --port 8000
```
سپس `http://127.0.0.1:8000/admin` را باز کنید.

## نکته‌ی امنیتی مهم

بررسی اعتبار اشتراک وقتی برنامه **به سرور دسترسی دارد** کاملاً معتبر و غیرقابل‌دورزدن
است (سرور همیشه حرف آخر را می‌زند). اما برای تحمل قطعی موقت اینترنت، برنامه تا ۷۲ ساعت
(قابل تنظیم در `OFFLINE_GRACE_HOURS`) روی آخرین نتیجه‌ی موفقِ آنلاین کار می‌کند. این
مهلت آفلاین به‌صورت محلی (در `settings.db`) نگه‌داری می‌شود و امضای رمزنگاری‌شده ندارد
(به‌خاطر مشکل نصب پکیج `cryptography` در محیط توسعه) — یعنی یک کاربر با دانش فنی
می‌تواند نظری این فایل را دستی ویرایش کند تا مهلت آفلاین را تمدید کند. این یک محدودیت
شناخته‌شده است؛ راه‌حل قوی‌تر (امضای نامتقارن با کلید عمومی در کلاینت) در آینده قابل
اضافه‌شدن است اگر لازم شد.
